    How to install the Eclipse prerequisites

    Overview

    The main prerequisite to run the Eclipse Embedded CDT is Java. Generally Java is not available by default on most platforms, and requires a separate installation.

    Although not mandatory, but highly recommended, the second prerequisite is xpm (the xPack Package Manager), which is a Node.js portable application.

    Prerequisites

    The following section covers the common Windows/macOS/Linux platforms; for a wider range of platforms, see the Alternate solutions below.

    • Windows
    • macOS
    • GNU/Linux

    The recommended package is the latest version from the official Oracle OpenJDK page.

    Follow the Download link in the Oracle page, which currently points to jdk.java.net/14.

    In the Builds section, download the Windows/x64 .zip file.

    Extract the content into ...\AppData\Roaming\Oracle\jdk-14.0.1.

    To set the path, in a cmd.exe terminal, issue the following:

    set Path=%APPDATA%\Oracle\jdk-14.0.1\bin;%Path%
setx Path "%APPDATA%\Oracle\jdk-14.0.1\bin;%Path%"

    Check version:

    java --version

    It is possible to install multiple versions on the same system, and chose which one to use by controlling the Path.

    The recommended package is the latest version from the official Oracle OpenJDK page.

    Follow the Download link in the Oracle page, which currently points to jdk.java.net/14.

    In the Builds section, download the macOS/x64 .tar.gz file.

    cd ~/Downloads
tar xf openjdk-14.0.1_osx-x64_bin.tar.gz
xattr -d com.apple.quarantine openjdk-14.0.1_osx-x64_bin.tar.gz
sudo mv jdk-14.0.1.jdk/ /Library/Java/JavaVirtualMachines/

    Then add the location to the user PATH:

    export PATH=/Library/Java/JavaVirtualMachines/jdk-14.0.1.jdk/:${PATH}

    Check version:

    java --version

    On recent macOS systems, this might fail or trigger a security message. To overcome this, go to System Preferences → Security & Privacy → Allow.

    It is possible to install multiple versions on the same system, and chose which one to use by controlling the PATH.

    To make Java available to all applications, including GUI ones, like Eclipse, it is necessary to configure the user path via:

    sudo launchctl config user path /Library/Java/JavaVirtualMachines/jdk-14.0.1.jdk/:${PATH}

    For this to be effective, reboot.

    On most modern Linux distributions, OpenJDK is available as a package, and can be installed via the sytem package manager.

    For example on Ubuntu:

    sudo apt install --yes openjdk-14-jdk

    Or on CentOS:

    sudo yum -y install java-14-openjdk

    Check your system for the available versions.

    However, if you cannot find at least JDK 11, install the Oracle OpenJDK.

    The recommended package is the latest version from the official Oracle OpenJDK page.

    Follow the Download link in the Oracle page, which currently points to jdk.java.net/14.

    In the Builds section, download the Linux/x64 .tar.gz file.

    cd ~/Downloads
tar xf openjdk-14.0.1_linux-x64_bin.tar.gz
mkdir -p ${HOME}/opt
sudo mv jdk-14.0.1.jdk/ ${HOME}/opt/

    Then add the location to the user PATH:

    export PATH=${HOME}/opt/jdk-14.0.1.jdk/:${PATH}

    Check version:

    java --version

    For GUI applications it is also necesary to set the path in a way specific to the window manager.
